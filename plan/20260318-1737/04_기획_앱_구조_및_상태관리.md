# Flutter 앱 아키텍처 및 상태 관리 (MVVM + Riverpod)

## 1. 디렉토리 구조 (추천)
```text
lib/
├── core/
│   ├── theme/           # 앱 테마, 컬러(어두운 톤), 타이포그래피
│   ├── constants/       # 상수(위험도 MAX 값 등)
│   └── utils/           # 로깅, JSON 파싱 유틸
├── data/
│   ├── models/          # GameState, SceneNode, Choice
│   └── repositories/    # 로컬 저장소(Hive/SharedPreferences) 및 JSON 스토리 로더
├── domain/
│   ├── entities/        # 순수 비즈니스 로직 및 상태 연산
│   └── usecases/        # 선택지 실행, 시간 진행 등
├── presentation/
│   ├── common/          # 재사용 위젯 (타이프라이터 텍스트, 커스텀 버튼)
│   ├── main_menu/       # 시작 화면
│   ├── story/           # 핵심 스토리 UI (텍스트 + 선택지)
│   ├── investigation/   # 단서/인물 수첩 시트
│   └── providers/       # Riverpod Provider 모음 (가장 중요)
```

## 2. Riverpod 핵심 상태 관리
이 게임의 모든 상태는 `GameState` 객체 하나에서 불변성(Immutable)을 유지하며 관리됩니다. (Freezed 패키지 활용 적극 권장)

### 2.1. GameState 모델 (Freezed)
```dart
@freezed
class GameState with _$GameState {
  const factory GameState({
    @Default('scene_101') String currentSceneId,
    @Default(0) int timeElapsed,
    @Default(0) int dangerLevel,
    @Default([]) List<String> evidence,
    @Default({}) Map<String, int> trustMap,
    @Default(false) bool isDead,
  }) = _GameState;
}
```

### 2.2. 핵심 Provider
* `storyNodeProvider`: `currentSceneId`를 바탕으로 현재 렌더링해야 할 JSON Scene 데이터(`SceneNode`)를 반환.
* `gameStateProvider`: 현재 게임의 진행 상태(시간, 단서, 위험도)를 담고 있으며, 사용자의 선택(Choice)에 따라 `StateNotifier` 혹은 `Notifier`에서 상태를 업데이트함.

## 3. JSON 기반 스토리 노드 설계

스토리는 하드코딩하지 않고 로컬 `assets/story/chapter1.json` 등의 파일로 관리합니다.

### 3.1. SceneNode JSON 구조
```json
{
  "id": "scene_101",
  "text": "당신은 촛불이 흔들리는 서재로 들어선다. 아서 블랙우드는 바닥에 쓰러져 있다.",
  "background_image": "mansion_study_room",
  "speaker": "system",
  "choices": [
    {
      "text": "서재를 수색한다 (시간 소모)",
      "next_scene_id": "scene_102",
      "conditions": { "required_trust": null, "required_evidence": null },
      "effects": {
        "time_cost": 1,
        "danger_delta": 0,
        "trust_delta": {},
        "add_evidence": ["locked_door"]
      }
    },
    {
      "text": "누군가 쫓아간다 (위험 행동)",
      "next_scene_id": "scene_103",
      "effects": {
        "time_cost": 0,
        "danger_delta": 1,
        "add_evidence": []
      }
    }
  ]
}
```

이 구조를 가지면 코드 수정 없이 기획자(작가)가 JSON만 수정해도 게임이 굴러갑니다.
