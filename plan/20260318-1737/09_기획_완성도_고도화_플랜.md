# 폭풍 저택의 유언 - 완성도 고도화 기획안

기본적인 TDD 시나리오 엔진이 완성되었으므로, 실제 상용 게임 퀄리티에 근접하도록 **'시각/청각적 연출'**과 **'핵심 편의 기능'**을 고도화합니다.

## User Review Required
> [!IMPORTANT]
> 이 기획안에서 제안하는 3가지 고도화 방향 중, **어떤 로직을 가장 우선순위로 개발할지** 피드백해주시면 계획을 반영하여 `EXECUTION` 모드로 진입하겠습니다.

## Proposed Changes

### 1. UI/UX 및 시청각 연출의 고도화 (몰입도 극대화)
* **타이프라이터 효과 (Typewriter Effect)**: 텍스트가 한 번에 출력되지 않고 타자기 소리와 함께 한 글자씩 출력되게 하여 텍스트 어드벤처 특유의 긴장감을 조성합니다.
* **사운드 시스템 (BGM & SFX)**: JSON 노드 설계에 `bgm` 및 `sfx` 속성을 추가합니다.
  * 예: `scene_101` 진입 시 `bgm: thunder_storm`, `sfx: door_creak` 발동.
* **화면 흔들림 (Camera Shake)**: `danger_delta`가 양수(+)로 튀는 위험한 씬에서 강제로 화면을 흔들어 플레이어에게 본능적인 경각심을 줍니다.

### 2. 편의 기능 고도화 (게임 루프 보완)
* **저장 및 불러오기 (Save/Load)**: Riverpod의 `GameState`를 로컬 데이터베이스(`SharedPreferences` 또는 `Hive`)에 JSON 형태로 직렬화하여 저장합니다. 게임을 껐다 켜도 `currentSceneId`부터 이어서 플레이할 수 있습니다.
* **챕터 라우팅 (Chapter Routing)**: `chapter1.json`과 `chapter2.json`을 단일 `story_repository`에서 매끄럽게 불러와 노드 간 끊김 없이 1시간 분량이 이어지도록 구조를 묶어줍니다.

### 3. 스토리 콘텐츠 고도화 (볼륨의 디테일)
* `scene_day2_hub_morning`에서 파생되는 8개 장소(용의자 방, 온실, 거실, 서재 등) 노드를 좀 더 세밀하게 쪼개고, **다회차를 유도하는 꽝(함정) 노드**를 다수 배치하여 추리의 난이도를 높입니다.

## Verification Plan
### Automated Tests
* JSON 파서가 `bgm`, `sfx` 필드를 누락 없이 읽고 `GameState`에 병합하는지 단위(Unit) 테스트.
* Riverpod 상태 저장 로직이 정상 동작하는지 테스트.
### Manual Verification
* `StoryScreen`에서 선택지 클릭 시 타이프라이터 효과가 리셋되고 자연스럽게 다음 Scene 이미지가 페이드인 되는지 화면 검증.
